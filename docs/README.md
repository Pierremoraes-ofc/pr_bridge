# 🌉 Introdução ao pr_bridge

O **pr_bridge** não é apenas uma "ponte" comum para FiveM. É uma biblioteca monstruosa de abstração e utilitários que permite escrever seus recursos uma única vez e rodá-los perfeitamente em múltiplos ecossistemas sem qualquer reescrita de código.

## Por que usar o pr_bridge?

1. **Abstração Total**: Você pode focar na lógica do seu script, pois o `pr_bridge` descobre sozinho qual Framework, Inventário, Database ou Notify o servidor está usando.
2. **Utilitários Essenciais**: O bridge já vem com centenas de utilitários cruciais para a tela (DrawText), criação de Blips, streaming de objetos, tuning de veículos e até manipulação de cache em memória para economizar chamadas no servidor.
3. **Ferramentas de Dev (DevTools)**: Esqueça comandos espalhados; o `pr_bridge` possui raycast interativo (DevLaser), ferramentas para posicionamento de entidades e freecam/gizmo.

Nesta documentação, detalharemos cada função, parâmetro e retorno para que você tire proveito máximo da biblioteca.
